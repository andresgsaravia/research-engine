In Research Engine some areas are formatted according to the Markdown syntax with a bit of GitHub flavor and the addition of Mathjax for math rendering. Below you can find a quick guide to some of the most common edditing tasks. For a complete reference please follow the previous links.

Paragraphs and linebreaks and headers

Paragraphs are just lines of text separated with whitespace. You can safely use linebreaks inside a paragraph and they will be rendered as expected. For example:

Disasters in the sun; and the moist star,
Upon whose influence Neptune's empire stands,
Was sick almost to doomsday with eclipse.

becomes

Disasters in the sun; and the moist star,
Upon whose influence Neptune's empire stands,
Was sick almost to doomsday with eclipse.

Headers

There are two styles of headers. For first-level and second level headers you can use a line of equal signs and dashes respectively.

This is a first-level header
============================


This is a second-level header
-----------------------------

Another option is to start the line with a hash character indicating the level of the header

# This is a first-level header
## This is a second-level header
### This is a third-level header
...

and so on to level 6. You may optionally “close” the headers with any number of hash characters. This is purely cosmetic in case you think it looks better.

# This is a first-level header #
## This is a second-level header ##
### This is a third-level header ###################

You can produde a Table of contents from your headers by writing [TOC] in the place you want it to appear.

Bold and Italics

Asterisk (*) and underscores (_) are used interchangeably as indicators of italics (or emphasis). If you use double asterisks (**) or underscores (__) you will get bold font. For example 

*Italic*
**Bold**
This is an *inline italic* with multiple words. Same with **bold words**.

You can still use * safely if you surround it with whitespace

is rendered as

Italic
Bold
This is an inline italic with multiple words. Same with bold words.

You can still use * safely if you surround it with whitespace

Lists

Unordered lists use asterisks (*) pluses (+) and hyphens (-) interchangeably as list markers. For example

* Red
* Green
* Blue

is equivalent to

+ Red
+ Green
+ Blue

and to

- Red
- Green
- Blue

All three get rendered as

  • Red
  • Green
  • Blue

Orered lists use numbers followed by periods. For example

1. Mexico
2. Tegucigalpa
3. Buenos Aires

becomes

  1. Mexico
  2. Tegucigalpa
  3. Buenos Aires

Notice that the actual numbers used to generate the list have no impact in the generated output. So you would get the same output if you write instead

18. Mexico
1. Tegucigalpa
25. Buenos Aires

This could generate some unintentional lists. To avoid this problem you will need to escape the period after the number. For example to generate the paragraph

1986. What a great season.

you need to write

1986\. What a great season.

Tables

To make a table you need to write something like

First Header  | Second Header
------------- | -------------
Content Cell  | Content Cell
Content Cell  | Content Cell

And it will be rendered as

First Header Second Header
Content Cell Content Cell
Content Cell Content Cell

See here for more options.

Code

To indicate a span of code wrap it with backtick quotes (`). This produces inline code, for example

To define a function in Common Lisp use the `(defun ...)` construct.

produces

To define a function in Common Lisp use the (defun ...) construct.

You have several options to produce a block of code. One option is to indent each line by 4 spaces or 1 tab 

    def hello(name = None):
        if name:
            print "Hello " + name
        else:
            print "Hello stranger"
        return

Another option is to fence the code with tildes 

~~~
def hello(name = None):
    if name:
        print "Hello " + name
    else:
        print "Hello stranger"
    return
~~~

or backquotes

```
def hello(name = None):
    if name:
        print "Hello " + name
    else:
        print "Hello stranger"
    return
```

all examples are rendered as

def hello(name = None):
    if name:
        print "Hello " + name
    else:
        print "Hello stranger"
    return

You can have syntax highlighting for some languages by adding the name of the language preceded by :::

```
:::python
def hello(name = None):
    if name:
        print "Hello " + name
    else:
        print "Hello stranger"
    return
```

results in

Quotes

Quotation is handled in email-style with the > sign. If you want to make a line appear as a quote you just prefix it with >. If the quote contains several lines you need to quote them all. Remember that quotes can be nested. Here is an example: 

> This line will be quoted.
> You need to quote each line individually
>
> > This is a nested quote
>

Links

There are two styles of links: inline and reference. In both styles the link text is delimited by [square brackets] The text in the square brackets is the text that will be shown. For inline links you follow them by a set of regular parentheses indicating the URL where you want the link to pint, for example

My favorite search engine is [DuckDuckGo](https://duckduckgo.com/ "Optional title here").

is transformed to

My favorite search engine is DuckDuckGo.

In reference links the first set of square brackets is followed by a second indicating a label to identify the link: 

But [Google][id] is my second favorite.

Then anywhere in the document you define the link label

[id]: https://www.google.com/ "Optional title here"

This produces the expected output:

But Google is my second favorite.

You can leave out the label in a reference link and the link text will be used as a label. For example 

I highly recommend you reading [Cory Doctorow][]. He's a terrific writter.
	      
	      [Cory Doctorow]: http://craphound.com/ "Doctorow's blog"

Notice that you can have spaces in the text link.

Finally, you can also have an URL transformed automatically into a link if you surround it with angle brackets:

This will be linked automatically: <http://example.com>

becomes

This will be linked automatically: http://example.com

You can have a doi autolinked by writing it in the following syntax:

This is a link to a journal article doi:10.1038/nmat2315

which will be transformed into a link like this

This is a link to a journal artile doi:10.1038/nmat2315

You can use "doi", "DOI" or "Doi" with the same result

Images

The syntax for images resembles the link syntax. You just need to prefix the first set of square brackets with an exclamation mark. It can be done in both inline and reference styles. 

![Alt text](http://domain.com/my_image.png "Optional title")


![Another image][myid]

[myid]: http://domain.com/image2.png "Also an optional title here"

Escaping special characters

Markdown allows you to use backslash escapes to generate literal characters which would otherwise have special meaning in Markdown’s formatting syntax. To do this you just need to escape it with a backslash:

\*literal asterisks\*

Markdown provides backslash escapes for the following characters:

\   backslash
`   backtick
*   asterisk
_   underscore
{}  curly braces
[]  square brackets
()  parentheses
#   hash mark
+   plus sign
-   minus sign (hyphen)
.   dot
!   exclamation mark

Mathematics

We use Mathjax as the display engine for mathematics using $\LaTeX$ syntax. The delimiters are $ ... $ for inline math and $$ ... $$ for displayed equations. This is an example: 

This equation: $e^{i\pi + 1} = 0$ is one of my favorites, but this one
$$ - \frac{\hbar^2}{2 m}\frac{\partial^2 \psi(x)}{\partial x^2} = E \psi(x) $$
looks far more impressive.

which is displayed as:

This equation: $e^{i\pi + 1} = 0$ is one of my favorites, but this one
$$ - \frac{\hbar^2}{2 m}\frac{\partial^2 \psi(x)}{\partial x^2} = E \psi(x) $$ looks far more impressive.

You can right-click at the rendered equation and copy the $\LaTeX$ code to your clipboard. Notice that equations will not be rendered inside code blocks.

If you want to write literal dollar signs \$ you will need to escape them with a backslash \$.
